# 20260330-slice_3b.md
## Slice 3B: Live Multi-Turn Spoken Continuity

**Date:** 2026-03-30
**Status:** Drafted from approved Slice 3 re-scope
**Depends on:** Slice 0 (Verified), Slice 1 (Verified), Slice 2 (Verified), Slice 3A (Deterministic continuity gate)

**Slice 3B goal:** Preserve the original Slice 3 product direction by proving a thin live spoken continuity gate on top of the Slice 3A continuity substrate.
Two spoken turns occur in one live session, and the second spoken turn demonstrably uses context from the first.

The completed loop after Slice 3B:
```
hardware-aware startup
→ Slice 3A continuity substrate available
→ live spoken session opened
→ spoken turn 1: STT → cognition → response → TTS → artifact written
→ spoken turn 2: STT → cognition with turn 1 context injected → response → TTS → artifact written
→ live session ends cleanly
```

Slice 3B is a **thin live spoken integration gate**.
It reuses the Slice 3A continuity substrate and does not expand the memory or persistence design beyond the original Slice 3 intent.

---

## Why Slice 3B Exists

`ProjectVision.md` still requires the real voice-first product loop to work in the active runtime.
However, once continuity itself is proven deterministically in Slice 3A, the remaining question is narrower:

Can the already-defined continuity substrate survive two spoken turns in one live session?

That is the purpose of Slice 3B.

---

## Slice 3B Scope

Slice 3B is limited to proving the live spoken integration boundary:

1. **Shared live session across two spoken turns**
   - the spoken path reuses one continuity session instead of treating each turn as isolated

2. **Spoken turn continuity**
   - the second spoken turn receives explicit prior-turn context from the first

3. **Artifact continuity under live execution**
   - both spoken turns are recorded under the same session boundary

4. **Narrow live-path confirmation**
   - the existing Slice 1/2 spoken runtime path remains intact while proving the two-turn continuity gate

---

## Slice 3B Non-Goals

Slice 3B does **not**:
- redesign the Slice 3A session/artifact/write-policy model
- broaden into long-term memory, retrieval, summarization, or restart-safe reconstruction
- broaden into degraded-path redesign or recovery-architecture work
- reopen closed Slice 0–2 work beyond non-regression confirmation
- broaden into interruption or Slice 4 concerns

Live failures must remain explicit, but handling them is limited to confirming the existing runtime behavior is not regressed while proving this narrow gate.

---

## Sub-Slices

### 3B.1 — Spoken session reuse
Carry one continuity session across two spoken turns using the already-defined Slice 3A substrate.

### 3B.2 — Live two-turn spoken proof
Demonstrate in the active runtime that spoken turn 2 can use context from spoken turn 1.

### 3B.3 — Spoken-path non-regression confirmation
Confirm the established Slice 1/2 spoken runtime path remains intact while adding the narrow live continuity gate.

---

## Acceptance Criteria

Slice 3B is complete when all of the following are true:

1. one live session spans two spoken turns
2. spoken turn 1 completes through the existing voice path
3. spoken turn 2 completes in the same session and demonstrably uses turn 1 context
4. both spoken turns are recorded as artifacts under the same session boundary
5. the live proof stays within the Slice 3A continuity design rather than redefining it
6. existing Slice 1/2 spoken runtime behavior is not regressed by reference validation

Slice 3B is **not** complete because broader memory ambitions were added.
It is complete only when the already-proven continuity substrate works through a narrow live spoken two-turn gate.

---

## Design Assumptions Changed From Original Slice 3

Compared with `20260330-slice_3.md`, Slice 3B changes the role of the live test:

1. **The live multi-turn spoken path is no longer the primary architectural proof.**
   - That proof belongs to Slice 3A.

2. **The live path becomes an integration gate, not the design-definition gate.**
   - Slice 3B validates reuse of the continuity substrate under spoken execution.

3. **Scope remains intentionally thin.**
   - Slice 3B proves two spoken turns in one session with explicit continuity and nothing broader.

4. **The original Slice 3 product goal remains intact.**
   - The system still must support coherent repeated voice turns in one session.
   - The difference is proof order: deterministic continuity first, live spoken continuity second.

---

## Relationship to Slice 3A

Slice 3B assumes Slice 3A is already defined and accepted as the continuity substrate.
If continuity semantics, artifact semantics, or write-policy semantics are still unsettled, that work belongs in Slice 3A rather than here.
