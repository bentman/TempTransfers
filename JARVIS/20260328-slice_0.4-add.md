## Addendum: Subtask 0.4 — Vendor-Specific GPU Detection

**Date:** 2026-03-28
**Amends:** `20260328-slice_0.md`, Subtask 0.4
**Scope:** `backend/app/hardware/detectors/gpu_detector.py` only
**Status:** Replaces the original Subtask 0.4 implementation details in full. All other subtasks unchanged.

**Root cause of revision:** On the target host (NVIDIA RTX 3060, Windows), `Win32_VideoController.AdapterRAM` returned 4.0 GB against a known physical 12 GB. CIM `AdapterRAM` is not authoritative VRAM and must be removed from the VRAM path entirely. Research also confirmed that no single cross-vendor Python library provides reliable VRAM for NVIDIA, Intel, and AMD simultaneously. A vendor-specific adapter pattern is required.

---

## Revised Architecture

`detect_gpu()` is the single public function. It calls three vendor-specific internal adapters in priority order and returns the first positive GPU result under a unified output schema. Each adapter is fully guarded. If all adapters return nothing, `gpu_available` is `False`.

```
detect_gpu() → dict
    ├── _nvidia_adapter()   → authoritative VRAM via NVML/nvidia-smi/torch
    ├── _intel_adapter()    → identity + VRAM via OpenVINO or xpu-smi subprocess
    └── _amd_adapter()      → identity + VRAM via pyrsmi/amdsmi or rocm-smi subprocess
```

A **fourth pass** runs only if all three vendor adapters return no GPU, using onnxruntime provider hints as a last-resort identity signal. This pass never produces VRAM.

---

## Unified Output Schema

`detect_gpu()` returns a single dict conforming to this schema regardless of which adapter sourced the result:

```python
{
    "gpu_available":   bool,
    "gpu_name":        str | None,
    "gpu_vendor":      str | None,   # "nvidia" | "intel" | "amd" | "unknown"
    "gpu_vram_gb":     float | None, # None when no authoritative source exists
    "gpu_vram_source": str | None,   # "nvml" | "nvidia-smi" | "torch-cuda" |
                                     # "openvino" | "xpu-smi" |
                                     # "pyrsmi" | "rocm-smi" | None
}
```

**Rule:** `gpu_vram_gb` is `None` whenever `gpu_vram_source` is `None`.
`Win32_VideoController.AdapterRAM` is **never** used as `gpu_vram_gb`. It may be used for `gpu_name` identity only.

## NVIDIA Adapter

**File section:** `_nvidia_adapter()` in `gpu_detector.py`
**Authoritative VRAM sources in priority order:**

### Pass 1 — GPUtil (NVML binding, most reliable on Windows + Linux)

```python
try:
    import GPUtil
    gpus = GPUtil.getGPUs()
    if gpus:
        gpu = gpus[0]
        return {
            "gpu_available": True,
            "gpu_name": gpu.name,
            "gpu_vendor": "nvidia",
            "gpu_vram_gb": round(gpu.memoryTotal / 1024, 2),
            "gpu_vram_source": "nvml",
        }
except Exception:
    pass
```

GPUtil reads from NVML (same source as `nvidia-smi`). On this host this is the path that would have returned `12.0` GB correctly. This is now the primary path, not a fallback.

### Pass 2 — nvidia-smi subprocess (when GPUtil/NVML unavailable)

```python
import subprocess, sys

def _nvidia_smi_vram() -> float | None:
    try:
        result = subprocess.run(
            ["nvidia-smi", "--query-gpu=memory.total",
             "--format=csv,noheader,nounits"],
            capture_output=True, text=True, timeout=5
        )
        if result.returncode == 0:
            line = result.stdout.strip().splitlines()[0]
            mib = float(line.strip())
            return round(mib / 1024, 2)
    except Exception:
        pass
    return None
```

If `nvidia-smi` succeeds but GPUtil is absent, this is the authoritative fallback. `gpu_vram_source` = `"nvidia-smi"`.

### Pass 3 — torch.cuda (when nvidia-smi also unavailable)

```python
try:
    import torch
    if torch.cuda.is_available():
        props = torch.cuda.get_device_properties(0)
        vram_gb = round(props.total_memory / (1024 ** 3), 2)
        name = torch.cuda.get_device_name(0)
        return {
            "gpu_available": True,
            "gpu_name": name,
            "gpu_vendor": "nvidia",
            "gpu_vram_gb": vram_gb,
            "gpu_vram_source": "torch-cuda",
        }
except Exception:
    pass
```

`gpu_vram_source` = `"torch-cuda"`.

### Pass 4 — Identity only (NVIDIA GPU present, no VRAM source available)

If name/vendor is known from any NVIDIA signal (e.g. CIM name string contains "NVIDIA") but none of the above yielded VRAM:

```python
return {
    "gpu_available": True,
    "gpu_name": gpu_name_from_cim,   # identity string only
    "gpu_vendor": "nvidia",
    "gpu_vram_gb": None,
    "gpu_vram_source": None,
}
```

## Intel Adapter

**File section:** `_intel_adapter()` in `gpu_detector.py`

Intel GPU VRAM detection is harder than NVIDIA. Research confirmed that OpenVINO's `GPU_DEVICE_TOTAL_MEM_SIZE` property returns `UNSUPPORTED TYPE` on integrated Intel GPUs (documented in OpenVINO 2024 query device output). It is therefore not universally reliable. The adapter uses two passes, with a clear fallback to identity-only.

### Pass 1 — OpenVINO device query (best effort, not universally reliable)

```python
try:
    import openvino as ov
    core = ov.Core()
    devices = core.available_devices
    gpu_devices = [d for d in devices if d.upper().startswith("GPU")]
    if gpu_devices:
        dev = gpu_devices[0]
        full_name = core.get_property(dev, "FULL_DEVICE_NAME")
        vram_gb = None
        vram_source = None
        try:
            # GPU_DEVICE_TOTAL_MEM_SIZE is UNSUPPORTED on iGPU —
            # attempt it but treat failure as identity-only, not an error
            mem_bytes = core.get_property(dev, "GPU_DEVICE_TOTAL_MEM_SIZE")
            if isinstance(mem_bytes, (int, float)) and mem_bytes > 0:
                vram_gb = round(mem_bytes / (1024 ** 3), 2)
                vram_source = "openvino"
        except Exception:
            pass   # expected on iGPU; not a failure condition
        return {
            "gpu_available": True,
            "gpu_name": full_name or dev,
            "gpu_vendor": "intel",
            "gpu_vram_gb": vram_gb,
            "gpu_vram_source": vram_source,
        }
except Exception:
    pass
```

**Expected behavior by device type:**
- Intel Arc discrete (A770, B580, etc.): `GPU_DEVICE_TOTAL_MEM_SIZE` may return a value — treat as authoritative if > 0
- Intel iGPU (Iris Xe, UHD Graphics, etc.): `GPU_DEVICE_TOTAL_MEM_SIZE` returns `UNSUPPORTED TYPE` — identity-only, `vram_gb = None`

### Pass 2 — xpu-smi subprocess (discrete Arc only, Linux)

`xpu-smi` is Intel's discrete GPU management tool, available on Linux with Arc driver. It provides VRAM for Arc discrete but is absent on Windows and not installed for iGPU.

```python
import subprocess, json

def _xpu_smi_vram() -> tuple[str | None, float | None]:
    try:
        result = subprocess.run(
            ["xpu-smi", "discovery", "-j"],
            capture_output=True, text=True, timeout=5
        )
        if result.returncode == 0:
            data = json.loads(result.stdout)
            # structure varies by version; extract first device name + memory
            device = data.get("device_list", [{}])[0]
            name = device.get("device_name")
            mem_mb = device.get("memory_physical_size_byte", 0)
            if mem_mb > 0:
                return name, round(mem_mb / (1024 ** 3), 2)
    except Exception:
        pass
    return None, None
```

If xpu-smi succeeds: `gpu_vram_source` = `"xpu-smi"`.

### Pass 3 — Identity-only from CIM (Windows, no driver tools available)

On Windows where OpenVINO is absent and xpu-smi doesn't exist, fall back to CIM name detection for identity:

```python
import subprocess, sys

def _cim_gpu_names() -> list[str]:
    if sys.platform != "win32":
        return []
    try:
        result = subprocess.run(
            ["powershell", "-NoProfile", "-Command",
             "Get-CimInstance Win32_VideoController | Select-Object -ExpandProperty Name"],
            capture_output=True, text=True, timeout=8
        )
        if result.returncode == 0:
            return [l.strip() for l in result.stdout.splitlines() if l.strip()]
    except Exception:
        pass
    return []
```

Filter names containing "Intel" for Intel identity. `gpu_vram_gb = None`, `gpu_vram_source = None`.

## AMD Adapter

**File section:** `_amd_adapter()` in `gpu_detector.py`

AMD's authoritative VRAM path is `pyrsmi` (Linux, ROCm) or `rocm-smi` subprocess (Linux). On Windows, AMD ROCm is not supported — Windows AMD detection is identity-only from CIM.

### Pass 1 — pyrsmi (Linux + ROCm driver, authoritative)

`pyrsmi` is AMD's official Python binding to `amdsmi`/`librocm_smi64`. It requires ROCm to be installed.

```python
def _pyrsmi_detect() -> dict | None:
    try:
        from pyrsmi import rocml
        rocml.smi_initialize()
        try:
            count = rocml.smi_get_device_count()
            if count > 0:
                name = rocml.smi_get_device_name(0)
                mem_bytes = rocml.smi_get_device_memory_total(0)
                vram_gb = round(mem_bytes / (1024 ** 3), 2) if mem_bytes > 0 else None
                return {
                    "gpu_available": True,
                    "gpu_name": name,
                    "gpu_vendor": "amd",
                    "gpu_vram_gb": vram_gb,
                    "gpu_vram_source": "pyrsmi" if vram_gb else None,
                }
        finally:
            rocml.smi_shutdown()
    except Exception:
        pass
    return None
```

`smi_shutdown()` is always called via `finally` — this is required by the pyrsmi contract.

### Pass 2 — rocm-smi subprocess (Linux, ROCm driver, when pyrsmi unavailable)

```python
import subprocess, re

def _rocm_smi_vram() -> tuple[str | None, float | None]:
    try:
        result = subprocess.run(
            ["rocm-smi", "--showmeminfo", "vram", "--json"],
            capture_output=True, text=True, timeout=8
        )
        if result.returncode == 0:
            import json
            data = json.loads(result.stdout)
            # rocm-smi JSON schema: {card0: {"VRAM Total Memory (B)": "N"}}
            card = next(iter(data.values()), {})
            mem_bytes = int(card.get("VRAM Total Memory (B)", 0))
            name_result = subprocess.run(
                ["rocm-smi", "--showproductname"],
                capture_output=True, text=True, timeout=5
            )
            name = None
            if name_result.returncode == 0:
                for line in name_result.stdout.splitlines():
                    if "Card series" in line or "GPU" in line.upper():
                        name = line.split(":")[-1].strip()
                        break
            vram_gb = round(mem_bytes / (1024 ** 3), 2) if mem_bytes > 0 else None
            return name, vram_gb
    except Exception:
        pass
    return None, None
```

`gpu_vram_source` = `"rocm-smi"` if VRAM obtained.

### Pass 3 — Identity-only from CIM (Windows) or name inference (Linux without ROCm)

On Windows, AMD ROCm is not supported. AMD GPU identity is detected from CIM name strings containing "AMD" or "Radeon". `gpu_vram_gb = None`, `gpu_vram_source = None`.

On Linux without ROCm, `/sys/class/drm/` can sometimes yield device name but not reliable VRAM. Identity-only path applies.

```python
# Linux: check /sys/class/drm for amdgpu devices (identity only)
import os, pathlib

def _sysfs_amd_name() -> str | None:
    try:
        drm = pathlib.Path("/sys/class/drm")
        for card in sorted(drm.glob("card*")):
            vendor_file = card / "device" / "vendor"
            if vendor_file.exists():
                vendor = vendor_file.read_text().strip()
                if vendor == "0x1002":   # AMD PCI vendor ID
                    label_file = card / "device" / "label"
                    if label_file.exists():
                        return label_file.read_text().strip()
                    return "AMD GPU (sysfs)"
    except Exception:
        pass
    return None
```

## Provider Hint Fallback (Pass 4 — no vendor adapter matched)

Runs only when all three vendor adapters return no result. Identity-only, no VRAM.

```python
try:
    import onnxruntime as ort
    providers = ort.get_available_providers()
    PROVIDER_VENDOR_MAP = {
        "CUDAExecutionProvider":  ("nvidia", "NVIDIA (provider hint)"),
        "ROCMExecutionProvider":  ("amd",    "AMD ROCm (provider hint)"),
        "DmlExecutionProvider":   ("unknown","DirectML GPU (provider hint)"),
        "CoreMLExecutionProvider":("apple",  "Apple GPU (provider hint)"),
    }
    for p in providers:
        if p in PROVIDER_VENDOR_MAP:
            vendor, name = PROVIDER_VENDOR_MAP[p]
            return {
                "gpu_available": True,
                "gpu_name": name,
                "gpu_vendor": vendor,
                "gpu_vram_gb": None,
                "gpu_vram_source": None,
            }
except Exception:
    pass
```

## CIM Helper (shared Windows identity utility)

The CIM name query (PowerShell `Win32_VideoController`) is shared by all three vendor adapters for Windows identity detection when their primary tools are unavailable. It is a separate internal helper, not a standalone pass.

```python
def _cim_gpu_names() -> list[str]:
    """Windows only. Returns display adapter names for vendor inference.
    AdapterRAM from CIM is explicitly NOT queried — unreliable for VRAM."""
    if sys.platform != "win32":
        return []
    try:
        result = subprocess.run(
            ["powershell", "-NoProfile", "-Command",
             "Get-CimInstance Win32_VideoController "
             "| Select-Object -ExpandProperty Name"],
            capture_output=True, text=True, timeout=8
        )
        if result.returncode == 0:
            return [l.strip() for l in result.stdout.splitlines() if l.strip()]
    except Exception:
        pass
    return []
```

`AdapterRAM` is never queried. The function exists for name-only identity.

## VRAM Source Trust Hierarchy (summary)

| Source | Vendor | Trust | Platform |
|--------|--------|-------|----------|
| `nvml` (GPUtil) | NVIDIA | Authoritative | Windows + Linux |
| `nvidia-smi` subprocess | NVIDIA | Authoritative | Windows + Linux |
| `torch-cuda` | NVIDIA | Authoritative | Windows + Linux |
| `pyrsmi` | AMD | Authoritative | Linux (ROCm required) |
| `rocm-smi` subprocess | AMD | Authoritative | Linux (ROCm required) |
| `openvino` `GPU_DEVICE_TOTAL_MEM_SIZE` | Intel | Best-effort (unreliable on iGPU) | Windows + Linux |
| `xpu-smi` subprocess | Intel | Authoritative (Arc discrete only) | Linux |
| CIM `Win32_VideoController` name field | All | Identity only — name string, no VRAM | Windows |
| CIM `AdapterRAM` | — | **Explicitly banned from VRAM path** | — |
| onnxruntime provider hints | All | Identity only | Windows + Linux |
| sysfs `/sys/class/drm/` | AMD | Identity only (no reliable VRAM) | Linux |

The subtask is complete when all of the following pass:

1. `detect_gpu()` returns a dict with all five required keys: `gpu_available`, `gpu_name`, `gpu_vendor`, `gpu_vram_gb`, `gpu_vram_source`

7. No GPU: `gpu_available = False`, all other fields `None`

## Implementation Notes for Agent

- Each vendor adapter (`_nvidia_adapter`, `_intel_adapter`, `_amd_adapter`) is a separate private function within `gpu_detector.py`. They do not call each other.
- `detect_gpu()` calls them in order: NVIDIA → Intel → AMD → provider hints → no GPU. It returns on the first success.
- Every subprocess call uses `timeout=` (5–8 seconds). A timeout is treated the same as any other exception — caught, logged at DEBUG level, execution continues to next pass.
- `pyrsmi.smi_shutdown()` must always be called via `finally` when `smi_initialize()` succeeds, to avoid resource leaks.
- The CIM PowerShell invocation is used only for name identity and only on `sys.platform == "win32"`. It is never used for VRAM.
- Logging within each guarded block is at `DEBUG` level for expected-absent optional libraries, `WARNING` for unexpected failures in present libraries.
