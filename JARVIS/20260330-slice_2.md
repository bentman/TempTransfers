# 20260330-slice_2.md
## Slice 2: Spoken Response — Local TTS via Kokoro

**Date:** 2026-03-30
**Status:** Approved for implementation
**Depends on:** Slice 0 (Verified), Slice 1 (Verified)

**Slice 2 goal:** Extend the Slice 1 voice turn into a fully spoken response.
After cognition returns a text response, JARVIS speaks it back through local TTS.

The completed acceptance loop for Slice 2 is:
```
hardware-aware startup
→ local STT transcription
→ shared cognition + personality
→ text response (Slice 1)
→ local spoken response via Kokoro TTS  ← NEW
→ explicit degraded-mode when TTS unavailable
```

No wake word. No memory expansion. No UI work.

---

## Runtime Prerequisites

All Slice 1 prerequisites carry forward. Additionally:

1. **`espeak-ng` on host (Linux/macOS only)** — Kokoro requires it as a system
   phonemizer on Linux/macOS. On Windows, `misaki[en]` (bundled) replaces it.
   This must be confirmed absent on the target Windows host before live test.
2. **Audio output device present** — default system output device accessible via
   `sounddevice`. Checked explicitly in the live test prerequisite gate.
3. **Kokoro model present** — `models/tts/kokoro-v1.0/` must pass `verify_model()`
   before the live test proceeds. Satisfied by Sub-slice 2.1.

## Sub-Slice 2.2 — TTS Runtime Interface and Kokoro Local Runtime

### Objective
Define the TTS abstraction and implement `KokoroTTSRuntime` — the local runtime
that generates speech from text to a WAV file. Separate from playback.

### Files to Create

**`backend/app/runtimes/tts/base.py`**
`TTSBase` ABC:
```python
def synthesize(self, text: str, output_path: str) -> str: ...   # abstract; returns output_path
def is_available(self) -> bool: ...                              # abstract
```

**`backend/app/runtimes/tts/local_runtime.py`**
`KokoroTTSRuntime(TTSBase)`:
- constructor: `__init__(self, model_name: str = "kokoro-v1.0", voice: str = "af_bella", device: str = "cpu")`
- `is_available()`: attempts `from kokoro import KPipeline`; returns `True` if importable; never raises
- lazy pipeline init on first `synthesize()` call — not on construction
- `_ensure_pipeline()`:
  - resolves `local_dir` from `get_tts_model_entry(self.model_name)`
  - calls `ensure_model(hf_repo_id, local_dir)` via manager
  - initializes `KPipeline(lang_code='a', repo_id=local_dir, device=self.device)`
  - CUDA-to-CPU fallback on DLL load failure — same `_is_cuda_dll_error()` pattern
    used in STT; print `[TTS DEVICE] CUDA unavailable — falling back to cpu`
- `synthesize(text, output_path)`:
  - calls `self._ensure_pipeline()`
  - generates audio via `KPipeline`
  - writes 24kHz mono WAV to `output_path` using `soundfile`
  - returns `output_path`
  - raises explicit `RuntimeError` on synthesis failure

**`backend/app/runtimes/tts/tts_runtime.py`**
`select_tts_runtime(report: FullCapabilityReport) -> TTSBase | None`:
- constructs `KokoroTTSRuntime` with device preference from `report.profile.cuda_available`
- returns the instance if `is_available()` returns `True`
- returns `None` if TTS is unavailable — `None` is the degraded-mode signal
- never raises

`misaki[en]` is Kokoro's bundled English phonemizer for Windows (replaces system

## Sub-Slice 2.3 — Audio Playback

### Objective
Implement audio playback as a standalone utility — fully separate from TTS generation.
`sounddevice` and `soundfile` are already present from Slice 1.

### Files to Create

**`backend/app/runtimes/tts/playback.py`**

`has_output_device() -> bool`:
- returns `True` if at least one output device with `max_output_channels > 0` exists
- never raises; used by live test prerequisite check

`play_audio(audio_path: str) -> None`:
- raises `RuntimeError("play_audio: file not found: {audio_path}")` if file absent
- raises `RuntimeError("play_audio: no output device available")` if no output device
- reads WAV with `soundfile.read()`
- plays via `sounddevice.play()` + `sounddevice.wait()`
- blocking; returns only after playback completes

## Sub-Slice 2.4 — SPEAKING State Integration into Voice Service

### Objective
Extend `run_voice_turn()` to transition through `SPEAKING`, invoke TTS,
play the audio, and return to `IDLE`. Define explicit degraded-mode behavior
when TTS is unavailable.

### Files to Modify

**`backend/app/services/voice_service.py`** — extend `run_voice_turn()`:

After the existing `RESPONDING` / `get_response()` step and before `engine.transition(IDLE)`:

```
engine.transition(SPEAKING)
tts = select_tts_runtime(report)

if tts is None:
    print("[DEGRADED] TTS unavailable — text response only")
    engine.transition(IDLE)
    return response

tts_audio_path = str(Path(temp_dir) / "response.wav")
tts.synthesize(response, tts_audio_path)
play_audio(tts_audio_path)
engine.transition(IDLE)
return response
```

Failure handling (consistent with existing pattern in `run_voice_turn`):
- `tts.synthesize()` raises → `_fail("tts.synthesize", exc)`, re-raise
- `play_audio()` raises → `_fail("play_audio", exc)`, re-raise
- `tts is None` → degraded path is not a failure; turn ends in `IDLE` with text response returned

## Key Design Decisions

**`select_tts_runtime()` returns `None`, not raises.** TTS unavailability is a
degraded-mode condition, not an error. The caller handles `None` explicitly and
continues with text-only output. Turn ends in `IDLE`, not `FAILED`.

**`synthesize()` and `play_audio()` are separate functions.** TTS generation and
audio playback are independently testable, independently fallible, and independently
replaceable. A future streaming TTS path or different playback backend can be
substituted at either layer without touching the other.

**`verify_model()` becomes family-aware.** The STT artifact set (`model.bin`,
`config.json`, `vocabulary.json`) does not apply to Kokoro. Adding a `family`
parameter is the minimal change that keeps all existing STT callers intact while
extending to TTS. No callers need updating.

**Voice selection is not personality-driven in Slice 2.** Voice is hardcoded to
`"af_bella"` as the default. Mapping `PersonalityProfile` dimensions (e.g.
`voice_energy`, `voice_pacing`) to Kokoro voice IDs is deferred to Slice 5
per `ProjectVision.md` (Presence and Persona).

**CUDA-to-CPU fallback mirrors the STT pattern.** `KokoroTTSRuntime._ensure_pipeline()`
uses the same `_is_cuda_dll_error()` detection and explicit fallback print as
`FasterWhisperSTT._ensure_model()`. Consistent behavior, consistent observability.
