# 20260330-slice_3.md
## Slice 3: Turn Continuity and Session Memory

**Date:** 2026-03-30
**Status:** Approved for implementation
**Depends on:** Slice 0 (Verified), Slice 1 (Verified), Slice 2 (Verified)

**Slice 3 goal:** Make the voice turn repeatable and coherent across a session.
Each turn is persisted as an artifact. Prior turn context is passed explicitly into cognition.
The second spoken turn in a session demonstrably references context from the first.

The completed loop after Slice 3:
```
hardware-aware startup
→ session opened, session ID assigned
→ turn 1: STT → cognition (no prior context) → response → TTS → artifact written
→ turn 2: STT → cognition (turn 1 context injected) → response → TTS → artifact written
→ session ends cleanly
```

No SQLite. No vector store. No semantic retrieval. No summarization.
Filesystem artifacts + in-process working memory only.

---

## Runtime Prerequisites

All Slice 2 prerequisites carry forward. Additionally:

1. **`data/turns/` directory writable** — turn artifacts are written under
   `data/turns/<session_id>/`. Directory created automatically on first write.
2. **`data/sessions/` directory writable** — session state file written on open/close.
3. **Ollama stable across multiple turns** — transient Ollama failures have been
   observed in prior live tests. The multi-turn live test must surface and report
   these explicitly rather than masking them.

3.1 → 3.2 → 3.3 → 3.4 → 3.5

## Sub-Slice 3.1 — Session Lifecycle Manager

### Objective
Introduce session identity. A session is opened at startup, carries a unique ID
across all turns, and is explicitly closed when the interaction ends.
`session_manager.py` is declared in `repo_tree.md` — this is its first implementation.

### Files to Create

**`backend/app/conversation/session_manager.py`**

```python
@dataclass
class Session:
    session_id: str          # uuid4 string
    started_at: str          # ISO timestamp
    ended_at: str | None     # None until closed
    turn_count: int          # incremented on each completed turn

class SessionManager:
    def open_session() -> Session: ...
        # generates session_id, records started_at, returns Session
        # writes session state to data/sessions/<session_id>.json

    def close_session(session: Session) -> Session: ...
        # records ended_at, writes final state to data/sessions/<session_id>.json
        # returns updated Session

    def increment_turn(session: Session) -> Session: ...
        # increments turn_count, updates session state file
        # returns updated Session
```

Session state file schema at `data/sessions/<session_id>.json`:
```json
{
  "session_id": "...",
  "started_at": "...",
  "ended_at": null,
  "turn_count": 0
}
```

All file writes use atomic write (write to temp, rename) to avoid partial state.
`open_session()` creates `data/sessions/` if it does not exist.
`SessionManager` holds no class-level state — all state is in the `Session` dataclass
and on disk. This keeps it stateless, testable, and deterministic.

### Objective
Define the canonical turn artifact and write it to disk after each completed turn.
`turn_artifact.py` and `storage.py` are declared in `repo_tree.md` — first implementations.

### Files to Create

    session_id: str

def write_turn_artifact(artifact: TurnArtifact) -> str:

Artifact path: `data/turns/<session_id>/<turn_id>.json`

## Sub-Slice 3.3 — Working Memory and Turn Context

### Objective
Add in-process working memory for the active session and inject prior turn context
into prompt assembly. Two concerns in one sub-slice because they are tightly coupled —
working memory exists to feed context into the prompt, and the prompt cannot carry
context without working memory.

### Files to Create

**`backend/app/memory/working.py`**

```python
@dataclass
class TurnSummary:
    turn_index: int
    transcript: str
    response_text: str

class WorkingMemory:
    def __init__(self, max_turns: int = 5): ...
        # max_turns: rolling window; oldest turn dropped when exceeded

    def add_turn(self, summary: TurnSummary) -> None: ...
        # append to internal list; trim to max_turns

    def get_context_turns(self) -> list[TurnSummary]: ...
        # return current turns in chronological order

    def clear(self) -> None: ...
        # reset to empty; called on session close
```

`WorkingMemory` is a pure in-process structure. No file I/O. Not persisted.
It is reconstructable from `list_session_turns()` + `read_turn_artifact()` if needed
in a later slice — but reconstruction is not required in Slice 3.

**`backend/app/cognition/prompt_assembler.py`** — extend existing function:

```python
def assemble_prompt(
    transcript: str,
    personality: PersonalityProfile,
    context_turns: list[TurnSummary] | None = None,
) -> str:
```

When `context_turns` is non-empty, prior turns are prepended as a structured
conversation history block before the current user turn. Format:
```
[System: <identity_summary>. Tone: <tone>.]

[Prior turns:]
User: <turn_0.transcript>
Assistant: <turn_0.response_text>
User: <turn_1.transcript>
Assistant: <turn_1.response_text>

User: <current transcript>
```

When `context_turns` is `None` or empty, behavior is identical to current Slice 1/2
implementation — no regression in the single-turn path.

## Sub-Slice 3.4 — Memory Write Policy

### Objective
Make write policy explicit and readable. Define exactly when a completed turn is
written to working memory and when it is not. Policy logic lives in one place,
not scattered across the voice service.

### Files to Create

**`backend/app/memory/write_policy.py`**

```python
@dataclass
class WriteDecision:
    should_write: bool
    reason: str          # always populated — explains the decision either way

def evaluate_write_policy(
    transcript: str,
    response_text: str,
    final_state: str,
    tts_available: bool,
) -> WriteDecision:
```

Policy rules for Slice 3 (all explicit, all readable):

| Condition | Decision | Reason |
|-----------|----------|--------|
| `final_state == "FAILED"` | skip | `"turn failed — not written to memory"` |
| `transcript.strip() == ""` | skip | `"empty transcript — not written to memory"` |
| `response_text.strip() == ""` | skip | `"empty response — not written to memory"` |
| otherwise | write | `"turn complete — written to memory"` |

`evaluate_write_policy` is a pure function — no side effects, no I/O.
It returns `WriteDecision` which is logged explicitly before acting on it.

**`backend/app/services/voice_service.py`** — update `run_voice_turn()` to:
1. Accept `session: Session` and `memory: WorkingMemory` as parameters
2. Pass `memory.get_context_turns()` to `assemble_prompt()`
3. After turn completes: call `evaluate_write_policy()`; print `[MEMORY] <reason>`
4. If `should_write`: call `memory.add_turn(TurnSummary(...))`
5. Build and write `TurnArtifact` to disk via `write_turn_artifact()`
6. Call `SessionManager.increment_turn(session)`
7. Return response text

The `TurnArtifact` is always written regardless of the write policy decision —
write policy governs working memory only, not artifact persistence.

Multi-turn live test:

```python
def test_multiturn_session_live() -> None:
    # Prerequisite checks (same pattern as prior live tests)
    # mic, STT model, TTS model, output device, Ollama

    report = run_profiler()
    personality = load_personality_profile("default")
    session = SessionManager.open_session()
    memory = WorkingMemory(max_turns=5)

    print(f"[SESSION] opened: {session.session_id}")

    # Turn 1 — establish context
    print("[TURN 1] starting — speak now")
    response1 = run_voice_turn(report, personality, session=session, memory=memory)
    session = SessionManager.increment_turn(session)
    print(f"[TURN 1] response: {response1}")
    print(f"[TURN 1] memory turns: {len(memory.get_context_turns())}")

    assert isinstance(response1, str) and len(response1.strip()) > 0
    assert len(memory.get_context_turns()) == 1
    assert len(list_session_turns(session.session_id)) == 1

    # Turn 2 — context injection
    print("[TURN 2] starting — ask about the previous turn")
    response2 = run_voice_turn(report, personality, session=session, memory=memory)
    session = SessionManager.increment_turn(session)
    print(f"[TURN 2] response: {response2}")
    print(f"[TURN 2] memory turns: {len(memory.get_context_turns())}")

    assert isinstance(response2, str) and len(response2.strip()) > 0
    assert len(memory.get_context_turns()) == 2
    assert len(list_session_turns(session.session_id)) == 2

    session = SessionManager.close_session(session)
    print(f"[SESSION] closed: {session.ended_at}")
    assert session.ended_at is not None
    assert session.turn_count == 2
```

The live test requires two spoken utterances. The second utterance should be phrased
to reference the first (e.g. "what did I just ask you?") so context injection is
demonstrable in the printed `[TURN 2] response`.

## Key Design Decisions

**`TurnArtifact` is always written; write policy governs working memory only.**
Every completed turn — including degraded-mode turns — produces a persistent artifact.
This is the authoritative record per `ProjectVision.md`. Write policy determines
whether a turn enters the in-process context that feeds future prompts, not whether
it is recorded.

**`WorkingMemory` is in-process only for Slice 3.**
No persistence, no reconstruction on restart. If the process restarts, context is
lost. This is acceptable for Slice 3. A restart-safe working memory built from
artifact history is a later slice concern.

**`prompt_assembler.py` signature is backward-compatible.**
`context_turns=None` is the default. All existing callers (Slice 1 and 2 tests,
single-turn paths) continue to work without modification.

**`run_voice_turn()` signature extends, not replaces.**
`session` and `memory` are new parameters. Making them keyword-only with defaults
(e.g. `session=None, memory=None`) preserves backward compatibility with Slice 1/2
tests that do not pass them. When `None`, artifact writing and memory updates are
skipped — the single-turn path is unchanged.

**Session files live in `data/sessions/`, turn artifacts in `data/turns/`.**
Matching `repo_tree.md` exactly. Both directories are mutable runtime state, not source.
Both must be covered by `.gitignore` if not already.
