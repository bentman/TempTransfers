## Slice 0: Hardware Profiler — Implementation Subtasks

**Date:** 2026-03-28
**Status:** Ready for implementation
**Scope:** `backend/app/hardware/` + `backend/app/core/capabilities.py` + `config/app/profiles.yaml` + `config/models/stt.yaml` (STT profile entries only) + tests

**Acceptance gate:** profiler is callable, returns a verified normalized capability profile and capability flags on the actual target runtime. No UI. No model loading. No voice invocation. Detection and emission only.

**Prior version reference:** v1 `Get-AvailableHardware` (PowerShell, Windows CIM/WMI), v3 `hardware.py` (Python, GPUtil + onnxruntime providers + psutil), v4 `hardware/service.py` (ported v3, simplified NPU), v5 `hardware_profiler.py` (cleanest Python, best NPU coverage, VRAM profile capping, fully typed). **V5 is the baseline** — V6 extends it with a normalized output contract and capability flags.

## Subtask 0.1 — Define Normalized Output Schema

**File:** `backend/app/core/capabilities.py`
**Purpose:** Single source of truth for the capability profile and capability flags types. All downstream systems import from here, not from `hardware/`.

### Implementation Details

Define two `dataclass` structures (or Pydantic models consistent with the rest of the backend):

**`HardwareProfile`** — raw detection output:
```
os: str                    # platform.system() → "Windows", "Linux", "Darwin"
arch: str                  # platform.machine() → "AMD64", "ARM64", "x86_64"
cpu_name: str              # processor name string
cpu_physical_cores: int
cpu_logical_cores: int
cpu_max_freq_mhz: float
gpu_available: bool
gpu_name: str | None
gpu_vendor: str | None     # "nvidia", "amd", "intel", "apple", "unknown"
gpu_vram_gb: float | None
cuda_available: bool
npu_available: bool
npu_vendor: str | None     # "intel", "qualcomm", "apple", "unknown"
npu_tops: float | None     # where detectable; None if unknown
memory_total_gb: float
memory_available_gb: float
device_class: str          # "desktop", "laptop", "constrained" — see 0.4
profile_id: str            # deterministic slug, e.g. "nvidia-cuda-desktop-32gb"
```

**`CapabilityFlags`** — derived boolean decisions:
```
supports_local_llm: bool
supports_gpu_llm: bool
supports_cuda_llm: bool
supports_local_stt: bool
supports_local_tts: bool
supports_wake_word: bool
supports_realtime_voice: bool
supports_desktop_shell: bool
requires_degraded_mode: bool
stt_recommended_runtime: str    # "faster-whisper", "whisper.cpp", "openvino-whisper", "cpu-whisper"
stt_recommended_model: str      # "whisper-large-v3-turbo", "whisper-small", "whisper-base", "whisper-tiny"
```

**`FullCapabilityReport`** — emitted object:
```
profile: HardwareProfile
flags: CapabilityFlags
```

Notes from prior versions:
- V5 used plain `dict` returns from `get_hardware_state()` — V6 uses typed dataclasses for inspectability and downstream contract stability.
- V3/V4 used async `get_hardware_state()` — detection in V6 is synchronous (no I/O); async wrapping is unnecessary here.

## Subtask 0.2 — OS and Architecture Detector

**File:** `backend/app/hardware/detectors/os_detector.py`

### Implementation Details

```python
import platform

def detect_os() -> dict:
    return {
        "os": platform.system(),          # "Windows", "Linux", "Darwin"
        "os_release": platform.release(),
        "arch": platform.machine(),       # "AMD64", "ARM64", "x86_64", "aarch64"
    }
```

Simple, no dependencies beyond stdlib. V1 used `$env:PROCESSOR_ARCHITECTURE` and `Win32_Processor.Architecture` — the Python equivalent via `platform.machine()` covers all targets.

**Failure mode:** never raises — `platform` is stdlib. Returns string "unknown" for any field that returns empty.

## Subtask 0.3 — CPU Detector

**File:** `backend/app/hardware/detectors/cpu_detector.py`
**Dependency:** `psutil`

### Implementation Details

Derived from V5 `_safe_cpu_info()` — cleanest implementation across versions:

```python
import platform
import psutil

def detect_cpu() -> dict:
    try:
        freq = psutil.cpu_freq()
        max_mhz = float(freq.max) if freq is not None else 0.0
    except Exception:
        max_mhz = 0.0
    return {
        "cpu_name": platform.processor() or "unknown",
        "cpu_physical_cores": int(psutil.cpu_count(logical=False) or 1),
        "cpu_logical_cores": int(psutil.cpu_count(logical=True) or 1),
        "cpu_max_freq_mhz": max_mhz,
    }
```

**Notes:**
- `platform.processor()` is used for the human-readable name; it can return empty on Linux — fallback to "unknown".
- V1 used `Win32_Processor` CIM — Python equivalent is `platform.processor()` + `psutil`.
- All fields individually guarded; no single failure should collapse the whole detector.

## Subtask 0.5 — CUDA Detector

**File:** `backend/app/hardware/detectors/cuda_detector.py`
**Dependencies:** `torch` (optional)

### Implementation Details

Derived from V5 `_detect_cuda_gpu()`:

```python
def detect_cuda() -> bool:
    try:
        import torch
        return bool(torch.cuda.is_available())
    except Exception:
        return False
```

Secondary check via onnxruntime providers if torch is unavailable:
```python
def detect_cuda_via_providers(providers: list[str]) -> bool:
    return any("CUDAExecutionProvider" in p for p in providers)
```

**Notes:**
- V3 had both checks; V5 simplified to torch-only. V6 keeps both for environments where torch is not installed but onnxruntime CUDA is present.
- `nvidia-smi` subprocess call (from V1) is not used in Python path — torch is the reliable gate.

## Subtask 0.6 — NPU Detector

**File:** `backend/app/hardware/detectors/npu_detector.py`
**Dependencies:** `openvino` (optional), `qaic` (optional)

### Implementation Details

Four detection paths, derived from V5 (most complete NPU coverage across versions):

**Apple Silicon:**
```python
def _detect_apple_npu() -> bool:
    try:
        import torch
        mps = bool(torch.backends.mps.is_available())
    except Exception:
        mps = False
    return mps and platform.system().lower() == "darwin" \
           and platform.machine().lower() in {"arm64", "aarch64"}
```

**Intel NPU (OpenVINO):**
```python
def _detect_intel_npu() -> bool:
    try:
        import openvino as ov
        core = ov.Core()
        return any("NPU" in d.upper() for d in core.available_devices)
    except Exception:
        pass
    # Provider hint fallback
    try:
        import onnxruntime as ort
        providers = [p.lower() for p in ort.get_available_providers()]
        return any("openvino" in p or "npu" in p for p in providers)
    except Exception:
        return False
```

**Qualcomm NPU:**
```python
def _detect_qualcomm_npu() -> bool:
    try:
        import qaic
        return qaic is not None
    except Exception:
        pass
    hints = (platform.machine() + " " + platform.processor()).lower()
    return "qcom" in hints or "qualcomm" in hints or "snapdragon" in hints
```

**Windows CIM extended scan** (derived from V1 `Get-AvailableHardware` PnP scan):
On Windows, supplement with `Win32_PnPEntity` scan for devices matching "NPU", "Neural Processing", "AI Accelerator", "Hexagon NPU", "VPU", filtering out audio/USB/HID devices. This catches NPUs that don't expose OpenVINO or qaic.

**Output:**
```python
def detect_npu() -> dict:
    # Returns: npu_available, npu_vendor, npu_tops (where known)
```

TOPS values where known from V1:
- Qualcomm Hexagon (Snapdragon X): 45 TOPS
- Intel AI Boost (Core Ultra 1xx series): 34 TOPS
- Unknown: None

## Subtask 0.7 — Memory Detector

**File:** `backend/app/hardware/detectors/memory_detector.py`
**Dependency:** `psutil`

### Implementation Details

Derived from V5 `_safe_memory_info()`:

```python
import psutil

def detect_memory() -> dict:
    try:
        vm = psutil.virtual_memory()
        return {
            "memory_total_gb": round(vm.total / (1024 ** 3), 2),
            "memory_available_gb": round(vm.available / (1024 ** 3), 2),
            "memory_percent_used": float(vm.percent),
        }
    except Exception:
        return {
            "memory_total_gb": 0.0,
            "memory_available_gb": 0.0,
            "memory_percent_used": 0.0,
        }
```

**Failure mode:** returns zero-valued dict. Never raises.

## Subtask 0.8 — Device Class Classifier

**File:** integrated into `backend/app/hardware/profiler.py` (not a separate detector)

### Implementation Details

Device class is derived from the assembled detection results, not from a separate detection pass. Logic:

```
if arch is ARM64 and os is Darwin → "laptop" (Apple Silicon)
if npu_vendor is "qualcomm" → "laptop" (Snapdragon)
if os is Windows and battery present → "laptop"
if memory_total_gb < 16 and not cuda → "constrained"
else → "desktop"
```

Battery detection on Windows:
```python
import psutil
battery = psutil.sensors_battery()
is_laptop = battery is not None
```

V1 used `$hardware.Platform` (x64 vs ARM64) for device class — V6 uses battery presence + arch + memory as a more complete signal.

## Subtask 0.9 — Capability Flag Derivation

**File:** integrated into `backend/app/hardware/profiler.py`

### Implementation Details

Flags are derived deterministically from the assembled `HardwareProfile`. No external calls at this stage.

```
supports_local_llm:     memory_total_gb >= 8
supports_gpu_llm:       gpu_available
supports_cuda_llm:      cuda_available
supports_local_stt:     memory_total_gb >= 4       # Whisper base/small always viable
supports_local_tts:     True                       # CPU TTS always viable at minimum
supports_wake_word:     memory_available_gb >= 2
supports_realtime_voice: cuda_available OR npu_available OR memory_total_gb >= 16
supports_desktop_shell: os in ["Windows", "Darwin", "Linux"]
requires_degraded_mode: memory_total_gb < 4 OR (not gpu_available AND memory_total_gb < 8)
```

**STT runtime and model selection** (informed by Advisor STT decision):
```
NVIDIA CUDA desktop:
    stt_recommended_runtime = "faster-whisper"
    stt_recommended_model   = "whisper-large-v3-turbo"

CPU-only (memory >= 16GB):
    stt_recommended_runtime = "faster-whisper"
    stt_recommended_model   = "whisper-small"

CPU-only (memory < 16GB):
    stt_recommended_runtime = "faster-whisper"
    stt_recommended_model   = "whisper-base"

Apple Silicon (Darwin + ARM64):
    stt_recommended_runtime = "whisper.cpp"
    stt_recommended_model   = "whisper-large-v3-turbo"

Intel NPU:
    stt_recommended_runtime = "openvino-whisper"
    stt_recommended_model   = "whisper-small"    # NPU ceiling — large-v3-turbo not yet supported

Qualcomm NPU:
    stt_recommended_runtime = "onnx-whisper"
    stt_recommended_model   = "whisper-small"

Degraded / constrained:
    stt_recommended_runtime = "faster-whisper"
    stt_recommended_model   = "whisper-tiny"
```

Note: Intel NPU model ceiling is `whisper-small` (OpenVINO 2025 NPU limitation — `large-v3-turbo` not supported on NPU path). This must be explicit in the flag, not silently resolved at runtime.

## Subtask 0.10 — Main Profiler

**File:** `backend/app/hardware/profiler.py`

### Implementation Details

Single callable entry point. Assembles all detectors and emits the `FullCapabilityReport`.

```python
from backend.app.core.capabilities import HardwareProfile, CapabilityFlags, FullCapabilityReport
from backend.app.hardware.detectors.os_detector import detect_os
from backend.app.hardware.detectors.cpu_detector import detect_cpu
from backend.app.hardware.detectors.gpu_detector import detect_gpu
from backend.app.hardware.detectors.cuda_detector import detect_cuda, detect_cuda_via_providers
from backend.app.hardware.detectors.npu_detector import detect_npu
from backend.app.hardware.detectors.memory_detector import detect_memory

def run_profiler() -> FullCapabilityReport:
    os_info   = detect_os()
    cpu_info  = detect_cpu()
    mem_info  = detect_memory()
    gpu_info  = detect_gpu()
    cuda      = detect_cuda() or detect_cuda_via_providers(gpu_info.get("providers", []))
    npu_info  = detect_npu()
    device_class = _classify_device(os_info, cpu_info, mem_info, npu_info)
    profile   = _build_profile(os_info, cpu_info, mem_info, gpu_info, cuda, npu_info, device_class)
    flags     = _derive_flags(profile)
    return FullCapabilityReport(profile=profile, flags=flags)
```

The profiler **does not**:
- load models
- present UI
- invoke voice runtimes
- write to `data/` or `cache/`

It detects and emits. That is its contract.

**`profile_id` generation:**
A deterministic slug built from detected fields:
```python
f"{gpu_vendor or 'cpu'}-{'cuda' if cuda else 'nocuda'}-{device_class}-{int(memory_total_gb)}gb"
# e.g. "nvidia-cuda-desktop-32gb", "cpu-nocuda-laptop-16gb", "intel-nocuda-desktop-64gb"
```

### Unit Test Requirements

Each detector unit test must:
- verify the return type and shape (all expected keys present)
- verify graceful failure when the underlying library is unavailable (mock import errors)
- verify no exception propagates to the caller under any mocked failure condition

Example pattern for `test_cpu_detector.py`:
```python
def test_cpu_detector_returns_expected_keys():
    result = detect_cpu()
    assert "cpu_name" in result
    assert "cpu_physical_cores" in result
    assert "cpu_logical_cores" in result
    assert "cpu_max_freq_mhz" in result

def test_cpu_detector_never_raises(monkeypatch):
    monkeypatch.setattr("psutil.cpu_count", lambda logical: None)
    result = detect_cpu()
    assert result["cpu_logical_cores"] >= 1   # fallback to 1
```

### Runtime Test Requirements

`backend/tests/runtime/test_profiler_live.py` must:
- call `run_profiler()` against the actual host
- assert `FullCapabilityReport` is returned (not None, not dict)
- assert `profile.os` is not empty
- assert `profile.memory_total_gb` > 0
- assert `flags.stt_recommended_runtime` is one of the known valid values
- assert `flags.stt_recommended_model` is one of the known valid values
- print the full profile and flags to stdout for manual inspection

This is the live validation evidence required for `CHANGE_LOG.md` and `SYSTEM_INVENTORY.md` entries.
