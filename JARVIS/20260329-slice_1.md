# 20260329-slice_1.md
## Slice 1: Minimal Voice Turn — Implementation Sub-Slices

**Date:** 2026-03-29
**Status:** Approved for implementation
**Depends on:** Slice 0 — Hardware Profiler (Verified)

**Slice 1 goal:** One complete voice turn in the real runtime:
explicit push-to-talk invocation → microphone capture → local STT transcription →
cognition with personality applied → visible text response printed to stdout.

No TTS. No wake word. No UI. No cloud calls.
Text output is the acceptance surface for this slice.

---

## Runtime Prerequisites

These must be true on the host before the Slice 1 live test can pass.
They are not implementation items — they are environmental gates.
The live test must check each one explicitly and fail with a named error if any is missing.

1. **Microphone present and accessible** — default system audio input device must be available
2. **Whisper model downloaded** — `whisper-large-v3-turbo` (or the model named by `flags.stt_recommended_model`) must be present in `models/` or resolvable by `faster-whisper`
3. **Ollama running with at least one model pulled** — `http://localhost:11434` must respond; at least one model must be available via `ollama list`

The `LLAMA_CPP_MODEL_PATH` setting will be `None` for Slice 1. This is expected. `LlamaCppLLM.is_available()` will return `False`, and the runtime selector will fall through to Ollama. This is correct behavior, not a failure.

---

## LLM Runtime Hierarchy (enforced from Slice 1 forward)

```
1. LlamaCppLLM     — primary; local model file; is_available() = False in Slice 1
2. OllamaLLM       — secondary; local Ollama server; active runtime in Slice 1
3. Cloud API       — tertiary; not wired until a later slice
```

This hierarchy must be encoded in `runtime_selector.py` from day one.
Switching the active runtime in a later slice must require only setting `LLAMA_CPP_MODEL_PATH`
in `.env` — no structural changes to any other layer.

## Sub-Slice 1.2 — LLM Runtime Interface and Selector

### Objective
Define the LLM abstraction and implement the runtime selector that enforces the
llama.cpp → Ollama hierarchy. Ollama is the active runtime for Slice 1.
The selector must be deterministic and fail closed.

### Files to Create

**`backend/app/core/settings.py`**
Reads from environment / `.env` via `os.environ.get()`. No external dependencies.
Exposes:
```python
OLLAMA_MODEL: str        # default: "phi3:mini"
OLLAMA_HOST: str         # default: "http://localhost:11434"
LLAMA_CPP_MODEL_PATH: str | None  # default: None
```

**`backend/app/runtimes/llm/base.py`**
`LLMBase` ABC:
```python
def complete(self, prompt: str) -> str: ...   # abstract
def is_available(self) -> bool: ...           # abstract
```

**`backend/app/runtimes/llm/local_runtime.py`**
`LlamaCppLLM(LLMBase)`:
- `is_available()`: returns `True` only if `LLAMA_CPP_MODEL_PATH` is set and the file exists
- `complete()`: raises `NotImplementedError("LlamaCppLLM: no model loaded — set LLAMA_CPP_MODEL_PATH")`
  if called when unavailable; actual llama.cpp inference wired in a later slice
- Must import-guard `llama_cpp` with `try/except ImportError`

**`backend/app/runtimes/llm/ollama_runtime.py`**
`OllamaLLM(LLMBase)`:
- `is_available()`: GET `{OLLAMA_HOST}/api/tags`; returns `True` if response is 200; timeout 3s; never raises
- `complete(prompt)`: POST `{OLLAMA_HOST}/api/generate` with `{"model": OLLAMA_MODEL, "prompt": prompt, "stream": false}`; returns `response["response"]`; raises explicit `RuntimeError("OllamaLLM: no response from {OLLAMA_HOST}")` on failure; timeout 60s
- Uses stdlib `urllib.request` or `httpx` — do not add `requests` if not already present

**`backend/app/routing/runtime_selector.py`**
`select_llm_runtime() -> LLMBase`:
```
1. try LlamaCppLLM; if is_available() → return it
2. try OllamaLLM; if is_available() → return it
3. raise RuntimeError("No LLM runtime available. Checked: LlamaCppLLM (LLAMA_CPP_MODEL_PATH not set or file missing), OllamaLLM (Ollama not reachable at {OLLAMA_HOST})")
```
No silent fallback. No default. Explicit failure with named paths checked.

### Dependencies to Add (`backend/requirements.txt`)

Add to `# LLM / AI` section:
```

httpx>=0.27
```

## Sub-Slice 1.3 — STT Runtime and Fixed-Duration Audio Capture

### Objective
Implement the STT abstraction, the `faster-whisper` local runtime, and fixed-duration
push-to-talk microphone capture. One utterance in, transcript string out.

### Files to Create

**`backend/app/runtimes/stt/base.py`**
`STTBase` ABC:
```python
def transcribe(self, audio_path: str) -> str: ...   # abstract
def is_available(self) -> bool: ...                 # abstract
```

**`backend/app/runtimes/stt/local_runtime.py`**
`FasterWhisperSTT(STTBase)`:
- constructor accepts `model_name: str` (e.g. `"large-v3-turbo"`) and `device: str` (e.g. `"cuda"` or `"cpu"`)
- `is_available()`: attempts `from faster_whisper import WhisperModel`; returns `True` if importable
- lazy model load on first `transcribe()` call — not on construction; model loading is slow and must not block startup
- `transcribe(audio_path)`: loads model if not loaded; returns full transcript as a single string; raises explicit `RuntimeError` if file not found or transcription fails

**`backend/app/runtimes/stt/stt_runtime.py`**
`capture_utterance(output_path: str, duration_seconds: int = 5) -> str`:
- records `duration_seconds` of audio from the default input device
- writes a 16kHz mono WAV to `output_path`
- returns `output_path`
- raises explicit `RuntimeError("capture_utterance: no input device available")` if no microphone detected
- uses `sounddevice` + `soundfile`
- `duration_seconds` default is 5 for Slice 1; no VAD, no silence detection

## Sub-Slice 1.4 — Conversation Engine, Cognition, and Voice Service Orchestration

### Objective
Wire the full Slice 1 turn: state machine → capture → transcribe → prompt assembly
with personality → LLM completion → response returned. This is the slice acceptance path.

### Files to Create

**`backend/app/conversation/states.py`**
`ConversationState` enum — exactly these 12 values from `ProjectVision.md`:
```python
BOOTSTRAP, PROFILING, IDLE, LISTENING, TRANSCRIBING,
REASONING, ACTING, RESPONDING, SPEAKING, INTERRUPTED, RECOVERING, FAILED
```

**`backend/app/conversation/engine.py`**
`ConversationEngine`:
- constructor: `__init__(self, report: FullCapabilityReport, personality: PersonalityProfile)`
- `state: ConversationState` — starts at `IDLE`
- `transition(self, new_state: ConversationState) -> None`: updates state; prints transition to stdout (`[STATE] IDLE → LISTENING`); raises `RuntimeError` if transition is to same state (no no-op transitions)
- No async. No threading. Deterministic synchronous transitions only for Slice 1.

**`backend/app/cognition/prompt_assembler.py`**
`assemble_prompt(transcript: str, personality: PersonalityProfile) -> str`:
- builds a prompt string that includes:
  - the personality `identity_summary` and `tone` as system context
  - the transcript as the user turn
- personality is an explicit parameter — it is never read from a global or ambient source
- returns a single string ready to pass to `LLMBase.complete()`

**`backend/app/cognition/responder.py`**
`get_response(prompt: str, llm: LLMBase) -> str`:
- calls `llm.complete(prompt)`
- returns the response string
- raises `RuntimeError("responder: empty response from LLM")` if result is empty or whitespace-only
- stateless; no memory, no session context in Slice 1

**`backend/app/services/voice_service.py`**
`run_voice_turn(report: FullCapabilityReport, personality: PersonalityProfile) -> str`:

Full Slice 1 turn sequence with explicit state transitions:

```
1. engine = ConversationEngine(report, personality)
2. engine.transition(LISTENING)
3. audio_path = capture_utterance(output_path=<data/temp/utterance.wav>, duration_seconds=5)
4. engine.transition(TRANSCRIBING)
5. stt = FasterWhisperSTT(report.flags.stt_recommended_model, device=<cuda if cuda_available else cpu>)
6. transcript = stt.transcribe(audio_path)
7. print(f"[TRANSCRIPT] {transcript}")
8. engine.transition(REASONING)
9. llm = select_llm_runtime()
10. print(f"[LLM RUNTIME] {type(llm).__name__}")
11. prompt = assemble_prompt(transcript, personality)
12. engine.transition(RESPONDING)
13. response = get_response(prompt, llm)
14. print(f"[RESPONSE] {response}")
15. engine.transition(IDLE)
16. return response
```

Any exception at any step must:
- transition engine to `FAILED`
- print `[FAILED] <step name>: <error message>`
- re-raise the exception

`data/temp/` must be created if it does not exist before writing the WAV file.
WAV file from the turn is retained for debugging (not deleted post-turn in Slice 1).

- `test_prompt_assembler_includes_personality`: assert `personality.identity_summary` or `personality.tone` present in assembled prompt

- `test_llm_selector_prefers_llama_cpp`: mock `LlamaCppLLM.is_available()` returning `True`; assert `select_llm_runtime()` returns `LlamaCppLLM` instance
- `test_llm_selector_falls_through_to_ollama`: mock `LlamaCppLLM.is_available()` returning `False`, `OllamaLLM.is_available()` returning `True`; assert returns `OllamaLLM`
- `test_llm_selector_raises_when_none_available`: mock both returning `False`; assert raises `RuntimeError`

**`backend/tests/runtime/test_slice1_sst_turn_live.py`**

Single live test — runs the full turn on actual hardware:

```python
def test_voice_turn_live():
    # prerequisite checks — fail with named error, not assertion error
    # 1. check microphone
    # 2. check FasterWhisperSTT.is_available()
    # 3. check OllamaLLM.is_available()
```
