## Slice 4 Goal

Make barge-in a first-class, traceable system behavior.

When JARVIS is speaking, the user can interrupt. Speech stops. The interruption is recorded in the turn artifact. The system transitions cleanly into `RECOVERING`. The caller can then invoke the next turn. No session corruption. No silent failure. No hang.

The completed behavior after Slice 4 (single-turn boundary preserved):

```
turn N: listen → transcribe → reason → respond → SPEAKING begins
→ user speaks during SPEAKING phase → barge-in accepted
→ speech stops immediately
→ engine transitions: SPEAKING → INTERRUPTED → RECOVERING
→ run_voice_turn() returns InterruptedTurnResult (not a normal response)
→ artifact for turn N updated with interrupted=True, interrupted_at=<timestamp>
→ caller invokes run_voice_turn() again for turn N+1
→ turn N+1 captures new utterance with session/memory context intact
```

**Scope note:** `run_voice_turn()` in the current repo is a single-turn function. Multi-turn continuity is achieved by calling it again, which Slice 3B already proved. Slice 4 does not introduce a continuous voice loop. It extends the single-turn boundary with interruption awareness. A higher-level loop is a future concern.

`INTERRUPTED` and `RECOVERING` already exist in `ConversationState`. Slice 4 gives them their first real implementation.

---

## Architectural Approach

### The core problem

`play_audio()` in `playback.py` currently calls `sd.play()` + `sd.wait()` — a blocking call. There is no hook to stop it externally. Simultaneously listening for a barge-in signal on the microphone while playing audio requires both to operate concurrently. That means threading.

### The chosen approach: threading-based concurrent listen + play

The barge-in listener runs on a background thread using `sd.InputStream` (a separate PortAudio stream from the TTS output). A shared `threading.Event` — the **interrupt flag** — is the sole communication channel. When the listener detects a valid barge-in signal, it sets the flag. `play_audio_interruptible()` polls the flag and calls `sd.stop()` on the output stream when it fires.

This is the minimal threading surface needed. It does not introduce async, coroutines, multiprocessing, or a new event bus.

### Why not VAD-based barge-in yet

VAD (voice activity detection) is the production-quality approach. However VAD introduces a new dependency (`webrtcvad` or `silero-vad`) and a new model/config surface. For Slice 4, a threshold-based energy detector on the input device is sufficient to prove the barge-in architecture. VAD can be introduced in a later slice as a drop-in replacement for the detection function inside `BargeInDetector` without changing the surrounding architecture.

### Architectural risk: `sounddevice` simultaneous input + output

PortAudio supports simultaneous input and output streams on most Windows hosts using separate `sd.InputStream` / `sd.play()` calls, but this has not been exercised in this repo yet. The 4.2 sub-slice validates the detector in isolation; the 4.5 live test is the authoritative proof of concurrent streams during real playback. If the host cannot open both simultaneously, the degraded path in 4.4 applies.

### Stream ownership and stop responsibility (critical)

- `BargeInDetector` owns its `sd.InputStream`. `detector.stop()` closes only that input stream via `stream.stop()` + `stream.close()`. It does **not** call `sd.stop()`.
- `play_audio_interruptible()` owns the output stream opened by `sd.play()`. On interruption it calls `sd.stop()` to halt the output stream, then returns `False`.
- After `play_audio_interruptible()` returns `False`, the voice service calls `sd.stop()` once more as a belt-and-suspenders clear before handing back to the caller.
- This matches the existing `sd.stop()` discipline already in `stt_runtime.py` and `playback.py`.

### Interruption result contract

`run_voice_turn()` currently returns `str`. Overloading `""` as an interruption signal is too close to the `responder`'s empty-response failure mode. Slice 4 introduces a named return type:

```python
from dataclasses import dataclass

@dataclass
class VoiceTurnResult:
    response: str | None          # None if interrupted
    interrupted: bool             # True if barge-in stopped playback
    interrupted_at: str | None    # ISO timestamp of interruption; None if not interrupted
```

`run_voice_turn()` return type changes from `str` to `VoiceTurnResult`. Existing callers in tests receive `result.response` for the string. This is a small breaking change to the function signature; all existing test callers are updated in 4.4.

## Sub-Slice 4.2 — Barge-In Signal Detector

### Objective

Implement `BargeInDetector` as a standalone, independently testable module. Validate it in isolation (detector thread starts, does not falsely trigger on silence, stops cleanly). Concurrent-stream proof during real playback is deferred to the 4.5 live test.

### Files to Create

**`backend/app/runtimes/stt/barge_in.py`**

```python
import threading
from typing import Any

class BargeInDetector:
    """Listens on the default input device for a barge-in energy signal.

    Runs a background sd.InputStream. When RMS energy exceeds threshold for
    min_trigger_frames consecutive audio blocks, sets interrupt_flag.

    Stream ownership:
    - Owns only the sd.InputStream it opens.
    - stop() closes that stream via stream.stop() + stream.close().
    - stop() does NOT call sd.stop() — that is the caller's responsibility
      for the output stream.

    Failure contract:
    - If the input stream cannot be opened, failed=True and failure_reason
      is set. interrupt_flag is never set. Callers must check detector.failed
      after start() before relying on the flag.
    """

    def __init__(
        self,
        interrupt_flag: threading.Event,
        *,
        threshold: float = 0.02,
        min_trigger_frames: int = 3,
        sample_rate: int = 16000,
        block_size: int = 1024,
    ) -> None: ...

    def start(self) -> None:
        """Open sd.InputStream and begin background detection."""

    def stop(self) -> None:
        """Stop and close the input stream. Does not call sd.stop()."""

    @property
    def failed(self) -> bool:
        """True if the stream could not be opened or detection crashed."""

    @property
    def failure_reason(self) -> str | None:
        """Named failure reason, or None if not failed."""
```

Implementation notes:

- Uses `sd.InputStream(callback=..., samplerate=..., blocksize=..., channels=1, dtype="float32")`.
- Callback computes `rms = float(np.sqrt(np.mean(indata ** 2)))`. Consecutive-frame counter increments above threshold, resets below. At `min_trigger_frames`, calls `interrupt_flag.set()`.
- `stop()` calls `self._stream.stop(); self._stream.close()`. Never calls `sd.stop()`.
- All exceptions in `start()` set `self._failed = True` and `self._failure_reason = str(exc)` — never propagate from background thread.

## Sub-Slice 4.3 — Interruptible Playback

### Objective

Add `play_audio_interruptible()` to `playback.py`. Validate that the function correctly distinguishes interrupted from normal completion using a real synthesized audio file, not import-only checks.

### Files to Modify

**`backend/app/runtimes/tts/playback.py`** — add alongside the existing `play_audio()`:

```python
def play_audio_interruptible(
    audio_path: str,
    interrupt_flag: threading.Event,
    *,
    poll_interval_seconds: float = 0.05,
) -> bool:
    """Play audio_path through the default output device.

    Returns True if playback completed normally.
    Returns False if interrupt_flag was set before completion.

    On interruption:
    - calls sd.stop() to halt the output stream
    - returns False immediately (does not raise)

    Raises RuntimeError if file not found or no output device.
    """
```

Implementation:

- `sf.read(audio_path)` → `sd.play(data, sample_rate)`.
- Poll loop: `while sd.get_stream().active: if interrupt_flag.is_set(): sd.stop(); return False; time.sleep(poll_interval_seconds)`.
- Natural completion: `sd.stop(); return True`.
- The existing `play_audio()` is unchanged.

### Objective

    engine.transition(IDLE)

spoken_normally = play_audio_interruptible(tts_audio_path, interrupt_flag)
detector.stop()
sd.stop()   # clear output stream unconditionally after interruptible playback

if not spoken_normally:
    interrupted_at = _now_iso()

    # Update the artifact already written by run_turn() for this turn.

            artifact.interrupted = True
            artifact.interrupted_at = interrupted_at
            artifact.final_state = ConversationState.INTERRUPTED.name
            write_turn_artifact(artifact)   # atomic overwrite via existing helper

    engine.transition(RECOVERING)
    print("[RECOVERING] session intact — caller may invoke next turn")

engine.transition(IDLE)
return VoiceTurnResult(response=response, interrupted=False, interrupted_at=None)

The default threshold (`0.02` RMS, 3 consecutive frames at 16kHz/1024-block) triggers reliably on normal speech within ~200ms.

## Key Design Decisions

**`VoiceTurnResult` replaces bare `str` return.** Empty string is indistinguishable from an empty-response failure. The named dataclass makes interruption an explicit, inspectable contract with no ambiguity.

**`BargeInDetector.stop()` does not call `sd.stop()`.** The detector owns only its `sd.InputStream`. The voice service owns the output stream and calls `sd.stop()` explicitly after `play_audio_interruptible()` returns. This separation prevents accidental cross-stream termination.

**Interruption entry is from `SPEAKING`, not `LISTENING`.** The engine guards `INTERRUPTED` and `RECOVERING` exits, not their entry. Any exit from `SPEAKING` remains legal (including `IDLE` for normal completion and `FAILED` for errors). Only the post-interruption path is guarded.

**Artifact update uses `list_session_turns()[-1]` + `read_turn_artifact()` + `write_turn_artifact()`.** No new storage helper. The existing atomic overwrite in `write_turn_artifact()` handles the update. The last entry from `list_session_turns()` is always the turn just completed by `run_turn()` since cognition finishes before `SPEAKING` begins.

**Live acceptance requires observed interruption.** A non-interrupted run of the live test is a test failure. This matches `ProjectVision.md`'s validation hierarchy: live runtime behavior is the primary acceptance surface, not just a best-effort check.

**Slice 4 does not introduce a continuous voice loop.** Multi-turn continuity after interruption is achieved by the caller invoking `run_voice_turn()` again — the same pattern as Slice 3B. A higher-level session loop is a future slice concern.

**VAD deferred.** Energy-threshold detection is sufficient to prove the barge-in architecture. `BargeInDetector._callback()` is the only place that needs to change when VAD is introduced later.
