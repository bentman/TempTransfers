# 20260330-slice_3a.md
## Slice 3A: Non-Interactive Session Continuity

**Date:** 2026-03-30
**Status:** Drafted from approved Slice 3 re-scope
**Depends on:** Slice 0 (Verified), Slice 1 (Verified), Slice 2 (Verified)

**Slice 3A goal:** Preserve the original Slice 3 product direction by proving session continuity deterministically before any live multi-turn spoken gate.
Each turn is persisted as an artifact. Prior turn context is passed explicitly into cognition. A second turn demonstrably uses context from the first under controlled, non-interactive conditions.

The completed loop after Slice 3A:
```
hardware-aware startup
→ session opened, session ID assigned
→ turn 1 enters the canonical cognition path under controlled non-interactive input
→ response produced → artifact written → continuity state updated
→ turn 2 enters the same cognition path with turn 1 context injected
→ response demonstrates continuity → artifact written
→ session ends cleanly
```

Slice 3A is the **deterministic acceptance gate** for Slice 3 continuity.
Microphone timing, speaker timing, live spoken-loop behavior, and other runtime variability are **not** part of Slice 3A acceptance.

The original Slice 3 restraint remains in force:
- no SQLite
- no vector store
- no semantic retrieval
- no summarization
- no restart-safe reconstruction
- no broader long-term memory architecture

Filesystem artifacts plus bounded session continuity state only.

---

## Why Slice 3A Exists

`ProjectVision.md` requires deterministic orchestration, explicit artifacts, and continuity across turns.
`SYSTEM_INVENTORY.md` shows the repository has verified Slice 0–2 coverage for hardware profiling, one-turn voice execution, and spoken response.
`CHANGE_LOG.md` shows live runtime behavior can vary across host/runtime conditions.

Therefore Slice 3 continuity should first be proven where the continuity model itself is under test, not microphone/speaker/live-runtime timing.

---

## Slice 3A Scope

Slice 3A defines and proves the continuity substrate:

1. **Session lifecycle**
   - explicit session identity
   - clean open / increment / close behavior
   - bounded session state tied to multiple turns

2. **Canonical turn artifacts**
   - each completed turn produces a persistent artifact
   - artifacts remain the authoritative record of what occurred in the session

3. **Bounded continuity context**
   - prior-turn context is retained explicitly for the active session
   - current-turn cognition receives only the bounded context needed for continuity

4. **Explicit write policy**
   - the rule for what enters active continuity context is deterministic and inspectable
   - artifact persistence remains separate from continuity-write decisions

5. **Controlled multi-turn proof**
   - turn 2 demonstrably uses turn 1 context through the same canonical cognition path
   - proof is non-interactive and controlled, not dependent on live spoken-loop variability

---

## Slice 3A Non-Goals

Slice 3A does **not**:
- prove live spoken multi-turn interaction
- redesign Slice 1 or Slice 2 runtime behavior
- introduce restart-safe session reconstruction
- introduce semantic retrieval or long-term memory systems
- expand memory architecture beyond the original Slice 3 intent
- broaden into interruption or Slice 4 concerns

## Sub-Slices

### 3A.1 — Session lifecycle
Define the session container used to group related turns under one continuity boundary.

### 3A.2 — Turn artifact persistence
Define the persistent per-turn record used as the authoritative session trace.

### 3A.3 — Active continuity context
Define the bounded prior-turn context carried forward within the active session.

### 3A.4 — Explicit continuity write policy
Define the deterministic rule that decides whether a completed turn enters active continuity context.

### 3A.5 — Deterministic continuity proof and non-regression
Prove under controlled, non-interactive conditions that a second turn can use first-turn context, while preserving verified Slice 0–2 behavior.

---

## Acceptance Criteria

Slice 3A is complete when all of the following are true:

1. a session can be opened, incremented across multiple turns, and closed cleanly
2. each completed turn produces a persistent artifact tied to that session
3. prior-turn continuity context is injected explicitly into the next turn’s cognition path
4. the rule governing active continuity writes is explicit, deterministic, and separately testable
5. a controlled, non-interactive two-turn proof demonstrates that turn 2 uses turn 1 context
6. Slice 0–2 verified behavior remains intact by non-regression reference

Slice 3A is **not** complete merely because live spoken turns appear to work.
Its acceptance is the deterministic proof of continuity itself.

---

## Design Assumptions Changed From Original Slice 3

Compared with `20260330-slice_3.md`, Slice 3A changes the proof order as follows:

1. **Continuity is proven before live spoken multi-turn behavior.**
   - The continuity substrate becomes the first acceptance gate.

2. **Deterministic acceptance comes before live runtime integration.**
   - Controlled proof validates the model without microphone/speaker timing variability.

3. **The original Slice 3 product goal is preserved.**
   - The goal remains coherent repeated turns within one session using explicit artifacts and explicit context.

4. **Live spoken continuity is deferred, not removed.**
   - It moves into Slice 3B as a thinner follow-on gate that reuses this substrate.
